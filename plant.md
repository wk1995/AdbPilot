# ADB 跨平台工具需求规划

## 项目目标

围绕 Android Debug Bridge（ADB）能力，构建一套可在 Windows、macOS、Linux 和 Android 平台使用的工具或组件，统一处理设备连接、应用管理、文件传输、日志诊断、脚本自动化和远程调试等常见场景。

## 目标用户

- Android 开发者：快速安装、调试、抓日志、管理多设备。
- 测试工程师：批量执行测试脚本、采集日志、导出报告。
- 运维和售后人员：诊断设备状态、远程协助、批量处理设备。
- 普通高级用户：备份文件、投屏、管理应用和设备信息。

## 核心需求

### 1. ADB 环境管理

- 自动检测本机是否安装 ADB。
- 支持配置自定义 ADB 路径。
- 支持内置或下载平台对应的 ADB 工具包。
- 显示 ADB 版本、运行状态和 server 状态。
- 支持重启 ADB server：`adb kill-server`、`adb start-server`。

### 2. 设备连接管理

- 展示 USB 和无线连接设备列表。
- 显示设备序列号、连接状态、系统版本、品牌、型号、CPU 架构、电量等信息。
- 支持多设备选择和默认设备设置。
- 支持无线调试连接：`adb connect host:port`。
- 支持断开无线设备：`adb disconnect`。
- 检测 unauthorized、offline、device not found 等异常状态并给出处理建议。

### 3. 应用管理

- 安装 APK，支持覆盖安装、降级安装、授权安装。
- 卸载应用，支持保留数据卸载。
- 查看设备已安装应用列表。
- 按包名搜索应用。
- 获取应用版本号、versionCode、安装路径、数据目录。
- 启动、停止、清除应用数据。
- 导出 APK。

### 4. 文件管理

- 支持电脑和设备之间上传、下载文件。
- 支持批量传输和目录传输。
- 支持常用目录快捷入口：Download、Pictures、Movies、DCIM、Android/data。
- 支持文件大小、修改时间、权限信息展示。
- 支持删除、重命名、创建目录。
- 对 Android 11 及以上受限目录给出权限和访问说明。

### 5. 日志和诊断

- 实时查看 logcat。
- 支持按包名、进程号、tag、日志级别过滤。
- 支持保存日志文件。
- 支持崩溃日志高亮和快速定位。
- 支持导出 bugreport。
- 支持查看 dmesg、getprop、settings、pm、am 等常用诊断信息。

### 6. Shell 命令执行

- 提供交互式 shell。
- 支持常用命令模板。
- 支持命令历史记录。
- 支持多设备并行执行命令。
- 支持脚本批量执行和结果导出。

### 7. 屏幕和输入控制

- 截屏并保存到本机。
- 录屏并导出。
- 模拟点击、滑动、输入文本、按键事件。
- 支持常用按键：Home、Back、Menu、Power、Volume。
- 后续可扩展投屏和远程控制能力。

### 8. 自动化任务

- 支持任务编排：连接设备、安装应用、清理数据、启动应用、采集日志。
- 支持批量设备执行。
- 支持失败重试、超时控制和结果汇总。
- 支持导入导出任务配置。

### 9. 平台适配

#### Windows

- 处理 ADB 驱动、USB 授权、环境变量和路径空格问题。
- 支持 `.exe` 形式的 ADB。
- 提示常见驱动安装问题。

#### macOS

- 兼容 Apple Silicon 和 Intel。
- 处理执行权限、Gatekeeper 和 Homebrew 安装场景。
- 支持 `/usr/local/bin`、`/opt/homebrew/bin` 等常见路径检测。

#### Linux

- 处理 udev 规则、设备权限和 adb server 权限问题。
- 支持常见发行版包管理器安装提示。
- 避免要求用户长期使用 root 运行。

#### Android

- 作为移动端工具时，优先支持无线调试连接。
- 兼容 Android 11+ 无线调试配对流程。
- 在权限受限场景下给出明确说明。

## 非功能需求

- 跨平台命令封装，避免业务层直接拼接系统命令。
- 所有 ADB 调用需要有超时、错误码和 stderr 捕获。
- 对中文路径、空格路径和特殊字符路径做兼容。
- UI 或 CLI 输出应明确显示当前目标设备。
- 敏感信息脱敏，例如设备序列号、账号、Wi-Fi 信息。
- 日志、报告和任务结果可导出。
- 关键功能需要单元测试和平台兼容测试。

## MVP 范围

第一阶段优先完成以下能力：

1. ADB 路径检测和版本显示。
2. 设备列表展示和设备状态识别。
3. APK 安装、卸载、启动、停止。
4. 文件 push/pull。
5. logcat 实时查看和保存。
6. 截屏。
7. 基础 shell 命令执行。

## 后续路线

### 第二阶段

- 无线调试连接和配对。
- 多设备批量操作。
- 应用列表和应用详情。
- bugreport 导出和解析。
- 自动化任务配置。

### 第三阶段

- 投屏和远程控制。
- 性能监控：CPU、内存、网络、帧率。
- 测试报告生成。
- 插件化命令模板。
- 团队共享配置。

## 风险点

- 不同 Android 版本对目录和权限的限制差异较大。
- Windows USB 驱动和厂商设备兼容问题较多。
- 无线调试在 Android 版本和厂商 ROM 上行为不完全一致。
- 多设备并发操作需要严格区分设备序列号，避免误操作。
- 长时间日志采集和大文件传输需要处理性能和中断恢复。

## 建议技术设计

- 抽象 `AdbClient`：负责命令执行、超时、错误处理。
- 抽象 `Device`：封装序列号、状态和设备属性。
- 抽象 `PlatformAdapter`：处理 Windows/macOS/Linux/Android 的差异。
- 抽象 `TaskRunner`：处理批量任务、重试、并发和报告。
- 保留 CLI 能力，后续可在此基础上扩展桌面端、Web 端或移动端 UI。
